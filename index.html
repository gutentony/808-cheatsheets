⌨️CLI 進階參數
常用旗標
--model <id>指定模型
-c / --continue恢復最近對話
--resume <n>恢復指定 session
--session-id多輪延續同一 session
--max-turns N限制最大輪數
--output-formattext / json / stream-json
權限 & 安全
--permission-modedefault / acceptEdits / plan / dontAsk
--allowedTools白名單指定工具
--disallowedTools黑名單禁止工具
--dangerously-skip-*跳過所有確認（= bypass）
平行開發
--worktree / -w獨立 git worktree
--tmux獨立 tmux session
Headless 模式
-p "指令"非互動執行（CI/CD 必備）
cat err | claude -pPipe 輸入分析
🔧環境變數（CLAUDE_CODE_ 前綴）
..USE_BEDROCK=1啟用 AWS Bedrock
..USE_VERTEX=1啟用 Vertex AI
ANTHROPIC_MODEL全域預設模型 ID
..DEFAULT_*_MODEL按角色指定模型
..MAX_TURNSHeadless 最大輪數
..MAX_OUTPUT_TOKENS單次最大 token（預設 32K）
..AUTOCOMPACT_PCT自動壓縮觸發 %（~83%）
..DISABLE_NONESSEN*關閉遙測（離線環境）
🧠Thinking 層級 & Effort 模式
Thinking（CLI 輸入）
think最低 · 快速回應
think hard中等思考
think harder深度思考
ultrathink最大 token 預算
Effort（/effort 指令）
low改名 / 錯字
medium日常開發（預設）
high多檔重構
max架構決策（10x token）

⚠️ 簡單問題用高 effort 不會更好，只會更慢更貴。選對等級 = 省錢又快。

📋CLAUDE.md 最佳實踐
寫法要訣
  • →控制在 < 200 行，超過後遵從率下降
  • →MUST / NEVER 語氣比 Prefer 有效 3x
  • →結構：WHAT → WHY → HOW
  • →每條規則問：移除會犯錯嗎？不會就刪
組織技巧
  • →根目錄 + 子目錄分層放 CLAUDE.md
  • →@path 引入額外檔，避免主檔過長
  • →/init 生成初版，commit 進 git 共維
  • →團隊統一規範放 managed-settings
🎯Context 管理策略
  • 💡70-80% 時手動 /compact（比自動壓縮品質好）
  • 💡Subagent：主 context 只收 ~2K 摘要，不會塞爆
  • 💡停用不用的 MCP（幾個就吃 30%+ 窗口）
  • 💡精確路徑比模糊描述 省 10x context
  • 💡30-45 分鐘 session 優於馬拉松 session
  • 💡prompt 加 use subagents → 投入更多算力

  • T1互動開發（你在旁邊）

  • T3批次 CI（全自動）
